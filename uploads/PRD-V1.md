## 1. 背景与目标

### 1.1 问题陈述
中小型电商商户（首批合作商：防辐射服装类目，**标品 + 定制**混合经营）日常售前咨询存在以下痛点：

1. 售前咨询量大、响应时段长（电商客服 7×16h），需 ~10 名客服才能维持淘宝/天猫/京东三店的接待
2. 客服人力成本高且逐年上升，新客服培训周期 2-4 周
3. 客服回复质量参差，新员工易答错关键参数与功效合规问题，给商户带来潜在合规风险
4. 高峰期（活动 / 上新 / 夜间）人工无法覆盖，错失成单

### 1.2 Why Now
- 国产大模型（豆包 / 通义 / 智谱 / Kimi / DeepSeek 等）能力可在售前商品咨询场景达到 90%+ 准确率
- 国产模型规避数据出境合规风险
- 淘宝 / 天猫 / 京东开放平台均提供消息推送 API，可全自动接入
- 已有真实合作商愿意作为一期试水共建用户

### 1.3 目标用户

**首批商户画像**
- 行业：防辐射服装（孕妇装、防辐射围裙等）
- 经营平台：淘宝 / 天猫 / 京东（多店铺）
- 商品结构：标品（标准款现货）+ 定制（量身 / 特殊规格）
- 当前人工客服：约 10 人，分早中晚班
- 客服关注：售前咨询为主（功效、面料、尺寸、定制、价格、物流）

**二期画像**（仅作设计参考，不在一期实现）
- 类目扩展：3C、美妆、家居、食品等
- 平台扩展：抖音、小红书、微信、独立站

### 1.4 一期产品目标
1. 替代首批合作商至少 **70%** 的售前人工客服工作量
2. AI 售前应答准确率 **≥ 90%**（沙盒评测集 + 真实抽样审核）
3. 客户感知不到 AI 痕迹（盲测：被识别为人工占比 **> 70%**）
4. 跑通"知识库 → 对话 → 成单 → 数据回流"的产品闭环
5. 跑通合规体系：PIPL 删除、品类禁忌话术、平台官方接入

### 1.5 业务目标
1. 验证产品可行性，为二期商业化（积分制）打基础
2. 沉淀首个行业模板（防辐射服装），加速后续同类目接入
3. 沉淀公共 KB / 评测集作为持续质量提升的资产

## 3. 用户与场景

### 3.1 商户侧 Persona

| Persona | 关心 | 高频使用 |
|---------|------|---------|
| P1 商户老板（决策者 / 付费方） | 成本、ROI、出事不出事 | 数据看板、成单归因、节省人力测算 |
| P2 商户运营（KB / 口吻） | AI 表现是否符合品牌、KB 是否完整 | 知识库、AI 人设、问题反馈、沙盒测试 |
| P3 商户客服（人工兜底 / 接管） | 什么时候要接、AI 在干什么 | 会话监控、转人工接管（在原平台如千牛） |

### 3.2 终端客户侧 Persona

| Persona | 行为 | 期望 | 处理 |
|---------|------|------|------|
| C1 普通买家 | 在商品详情页问参数 / 功效 / 价格 / 库存 | 快速、准确、像真人 | AI 全自动 |
| C2 高价值 / 批量买家 | 批量采购、定制咨询、议价 | 被认真对待、能直达决策人 | 触发数量阈值 / 关键词 → 转人工 |

### 3.3 关键场景叙事

**场景 A — 标品咨询**
- 客户："这件孕妇防辐射衣服可以洗吗？"
- AI："可以的呢，亲，我们家这款采用 XX 面料，建议 30°C 以下手洗，避免暴晒～具体洗护说明详情可看商品的洗护标签哦。"
- 命中：商品参数 KB + FAQ#洗护

**场景 B — 定制咨询（命中数量阈值）**
- 客户："我想定制 50 件，能给个团购价吗？"
- AI："好的亲，您要定制 50 件。这种批量定制需要专人为您对接最优方案，我马上请负责的同事联系您～"
- 触发：数量 50 ≥ 30 → 转人工 + 通知商户

**场景 C — 功效合规边界**
- 客户："穿这个能保证胎儿不畸形吗？"
- AI："亲，这款是用来减少日常电磁辐射暴露的辅助产品，不能替代医学防护。胎儿健康受多种因素影响，建议咨询医生哦～"
- 命中：超管禁忌话术（不能承诺医学功效）

**场景 D — 陪跑期**
- 客户："这个面料含金属吗？"
- AI 草稿："含哒亲～我们家面料里含 X% 银纤维，所以才能屏蔽辐射哦。"
- 商户客服审核：通过 → 发送
- 触发：陪跑满 7 天且采纳率 ≥ 80% → 可切全自动

## 4. 商户端功能详述

> 以下 10 个能力域均为一期 P0。每个能力域包含用户故事、功能要求、验收标准。

### 4.1 账户与团队 [P0]

**用户故事**
- 作为商户老板，我希望注册主账号并独立管理我的商家信息
- 作为商户老板，我希望邀请运营和客服作为子账号并分配权限
- 作为商户运营，我希望查看自己最近的操作日志自查误操作

**功能要求**
- 主账号注册（手机号 / 邮箱）+ 登录（含 2FA 可选）
- 子账号 CRUD（添加 / 停用 / 删除）
- 角色权限：管理员（全权限）、运营（KB + 人设）、客服（仅会话查看）
- 操作日志：记录关键操作（KB 改动、店铺启停、人设切换），按用户筛选

**验收标准**
- [ ] 商户老板可独立完成注册并首次登录
- [ ] 子账号无法访问超出自身角色的页面（前端隐藏 + 后端校验）
- [ ] 操作日志至少保留 90 天

---

### 4.2 店铺接入 [P0]

**用户故事**
- 作为商户运营，我希望通过 OAuth 一键授权店铺，无需复制粘贴 API Key
- 作为商户老板，我希望一键启停某店铺的 AI 服务
- 作为商户运营，我希望店铺断连立刻收到告警

**功能要求**
- 平台支持：淘宝 / 天猫 / 京东
- 接入流程：商户端"接入店铺" → 平台 OAuth → 回调拿 token → 入库
- 多店铺管理：单商户挂多店，每店独立配置
- 启停开关：店铺级 + 全局两档
- 状态监控：心跳检测 → 断连 / Token 过期 → 站内信 + 短信 / 钉钉告警

**验收标准**
- [ ] OAuth 完成后店铺出现在列表，状态"已接入"
- [ ] 暂停后该店消息不再触发 AI（默认走商户原客服）
- [ ] Token 过期前 7 天自动提醒续授权

**技术约束**
- 平台 token 加密存储（KMS / 国产同等方案）
- 平台 API 限流由我方做令牌桶 + 队列削峰

---

### 4.3 知识库 [P0]

**用户故事**
- 作为商户运营，我希望商品信息从平台自动同步入 KB，不必手填
- 作为商户运营，我希望批量导入卖点话术（PDF / Excel）
- 作为商户运营，我希望促销活动到期后自动失效

**功能要求**

| 子模块 | 内容 |
|------|------|
| 商品库 | 自动从平台 API 拉商品（SKU、价格、参数、主图）；商户手动补充卖点 / 对比话术；商品启用 / 禁用（不影响平台上架，只影响 AI 是否引用） |
| FAQ 库 | 问答对（多个相似问法 + 标准答案 + 命中条件）；启用 / 禁用 + 失效时间 |
| 促销规则 | 标题 / 规则文本 / 生效起止；过期自动失效 |
| 话术库 | 类型：开场白 / 引导成单 / 合规对比；必带话术 + 禁忌话术 |
| 文件导入 | PDF / Word / Excel / 截图 OCR → 自动切片 → 入 KB；商户可在切片预览页调整粒度 |
| 行业模板 | 一键导入"防辐射服装"行业模板（基础 FAQ + 禁忌话术） |
| 版本管理 | KB 整体快照 + 单条历史回滚 |

**验收标准**
- [ ] 接入店铺后 24h 内商品自动同步入 KB
- [ ] 促销规则到达 `expire_at` 后下次 AI 应答不再引用
- [ ] PDF 导入后商户可见切片预览并可调整
- [ ] 行业模板导入为追加，不覆盖商户已有内容
- [ ] KB 改动可回滚到任一历史版本

---

### 4.4 AI 人设（变量化模板）[P0]

**用户故事**
- 作为商户运营，我希望从几种预设品牌口吻里选一个，立刻看到示例
- 作为商户运营，我不想直接编写 prompt，但希望能填关键字段让 AI 像我们家
- 作为商户运营，我希望禁忌词被严格遵守

**功能要求**
- 品牌口吻库（来源：超管端预设），一期 ≥ 4 种：
  - 专业冷静 / 亲切活泼 / 高端简约 / 萌系软萌
- 选中后右侧实时预览 3-5 条示例对话
- 变量填空（不允许改 prompt 骨架）：
  - 身份名（默认"小 X 客服"）
  - 风格关键词（≤ 5 个）
  - 必带话术（≤ 3 条）
  - 禁忌词（≤ 20 个）
  - Emoji 用法（频繁 / 偶尔 / 不用）
  - 署名（结尾自动签）
- 切换前提示影响范围（"将影响 X 个店铺所有未来对话"）

**验收标准**
- [ ] 切换品牌口吻后 30s 内对新会话生效
- [ ] AI 输出含禁忌词时被拦截重生成，不会发出
- [ ] 所有变量字段都有合理默认值，新商户开箱可用

**非功能约束**
- 实时预览不消耗商户真实 token（用低成本模型预生成或缓存）

---

### 4.5 沙盒测试 [P0]

**用户故事**
- 作为商户运营，我希望接入店铺后立刻可以和 AI 试聊
- 作为商户运营，我希望看到 AI 用了哪条 KB 建立信任
- 作为商户运营，发现答错时一键标注

**功能要求**
- 模拟对话窗口（左用户输入，右 AI 回复）
- 每条 AI 回复展开可见：命中 KB 条目 + 用到的人设变量 + 模型版本
- 一键反馈：不会答 / 答错 / 答得不够好 / 违反品牌口吻
- 反馈进入"问题反馈"队列（详见 §4.6.5）

**验收标准**
- [ ] 沙盒回复延迟 < 3s（与真实链路同等）
- [ ] 命中 KB 条目带跳转链接（点击直达 KB 编辑页）
- [ ] 沙盒会话不计入真实数据看板

---

### 4.6 会话与监控 [P0]

#### 4.6.1 实时会话列表
- 显示客户头像 / 昵称、最后一条消息、状态（进行中 / 已结束）、source（AI / 人工 / 混合）
- 实时刷新（WebSocket 或长轮询）

#### 4.6.2 历史会话搜索
- 维度：时间范围、平台店铺、客户、关键词、是否成单、source
- 分页

#### 4.6.3 会话详情
- 时间线展示所有消息
- 每条消息显示 source 标签（AI / 人工 / 系统）
- AI 气泡：浅灰，可展开命中 KB
- 人工气泡：浅蓝
- 系统气泡：浅黄（如"AI 已切换为人工接管"）
- 会话尾部：是否成单 + 金额 + 归因（AI / 人工 / 混合）

#### 4.6.4 客户侧表现（关键合规决策）
- 客户在淘宝 / 天猫 / 京东聊天窗看到的是统一身份（"客服小 X"）
- 不区分气泡颜色
- 不显示"AI 正在输入"打字机效果
- 模拟人工节奏：500ms-3s 延迟（按消息长度），整段一次性发出

#### 4.6.5 反馈入口
- 在任一 AI 消息气泡上点击"反馈"
- 表单：
  - 类型（必选）：不会答 / 答错 / 答得不够好 / 违反品牌口吻
  - 正确答案（可选）
  - 标签（可多选）：商品、价格、定制、物流、合规
- 提交后进入商户端"问题反馈"列表 + 同步到超管训练样本池

**验收标准**
- [ ] `source` 字段在所有消息上必有，不可为空
- [ ] 反馈提交 24h 内可见于"问题反馈处理列表"
- [ ] 客户侧盲测：100 条 AI 回复被识别为人工占比 ≥ 70%

### 4.7 数据看板 [P0]

**用户故事**
- 作为商户老板，我希望首页就看到本月 AI 帮了多少单
- 作为商户老板，我希望看到节省了多少人力成本
- 作为商户运营，我希望看到 AI 答不上来的问题及时补 KB

**功能要求**
- 接待概览：接待量、回复率、平均响应时长（按日 / 周 / 月）
- 成单转化：会话→下单转化率，按 source 三段切分（AI 独立 / AI+人工 / 纯人工）
- 成单归因：本期 AI 推动成单数 + GMV
- 高频问题 TOP 20
- AI 答不上来的问题清单（一键加 KB）
- 节省人力测算：(AI 独立处理会话数 × 平均人工处理时长) / 客服月工时 = 等效节省人力

**验收标准**
- [ ] 数据 T+1 更新（凌晨重算）
- [ ] 节省人力测算公式可在帮助页查看
- [ ] "答不上来的问题"点击"加入 KB" → 跳 FAQ 新建（带预填）

**功能要求**

### 4.9 转人工 / 工作模式 [P0]

#### 4.9.1 转人工触发规则（可叠加）
- 关键词触发（多关键词 OR）
- 数量阈值（消息中提到 ≥ N 件 → 转）
- 关键词 + 数量组合
- 负面情绪检测（模型自带情感判断，置信度 ≥ 80% 触发）
- AI 不确定度（模型输出置信度 < 阈值）

#### 4.9.2 工作模式
- 全天 AI（所有消息 AI 回）
- 工时人工兜底（工时内：AI 草稿+人工确认；工时外：纯 AI）
- 高峰协助（仅高峰时段 AI 接，其他时段商户原客服处理）

#### 4.9.3 AI 兜底话术
- 默认值："亲～您的问题需要我们的专属顾问为您处理，请稍候，工作人员马上联系您～"
- 商户可改

#### 4.9.4 陪跑模式（半自动）
- 流程：客户消息 → AI 草稿 → 商户客服在商户端审核（修改 / 通过 / 拒绝）→ 平台发送
- 适用：商户上岗前的过渡期
- **切换至全自动的前提条件（同时满足）**：
  1. 沙盒测试通过 ≥ 200 条且准确率 ≥ 90%
  2. KB 完成度评分 ≥ 80%
  3. 培训完成（运营 / 客户成功标记）
  4. 陪跑期采纳率 ≥ 80%（连续 7 天）

#### 4.9.5 正式模式（全自动）
- 满足条件后商户老板可在商户端切换至全自动直发
- 紧急回退：管理员一键切回陪跑

**验收标准**
- [ ] 转人工触发规则可叠加，至少支持 3 条同时生效
- [ ] 触发转人工时，AI 给客户的过渡话术由商户配置
- [ ] 切换前不满足条件 → 按钮置灰，hover 提示原因
- [ ] 全自动模式下管理员可一键切回陪跑

---

### 4.10 通知与告警 [P0]

**功能要求**
- 告警类型：店铺断连、AI 服务异常、订单高峰（环比 X%）、差评激增
- 告警渠道：站内信（必有）+ 短信 / 钉钉（可配）
- 告警阈值可配

**验收标准**
- [ ] 店铺断连后 5 分钟内告警送达
- [ ] 商户可关闭某类告警

## 5. 超管端功能详述

> **超管端对商户保持黑盒**。所有功能仅我方运营 / 技术人员使用。

### 5.1 商户管理 [P0]
- 商户列表（行业、规模、店铺数标签）
- 准入审核：申请 → 资质审核 → 开通
- 暂停 / 关停（单商户 / 单店铺 / 全局粒度）
- 商户档案详情：合同、协议、合作模式、合作起止日期
- 套餐字段（一期为 free，二期切积分制时复用）

**验收标准**
- [ ] 关停商户后所有店铺自动暂停 AI
- [ ] 准入审核留痕（谁审、何时、决议）

---

### 5.2 模型管理 [P0]
- 模型清单（一期 2-3 个国产模型）
- 路由策略：
  - 按消息复杂度（短消息 / 常见 FAQ → 便宜模型；长上下文 / 复杂咨询 → 高级模型）
  - 按商户配置
  - 按场景（沙盒 vs 真实）
- 默认模型 + 灰度切换（按商户百分比）
- A/B 测试（同一商户分流，对比效果）
- 熔断降级：主模型超时 / 错误率高 → 自动切备用
- Prompt 缓存配置（命中策略、TTL）

**验收标准**
- [ ] 模型切换无需重启服务
- [ ] 熔断触发后 30s 内 100% 流量切到备用
- [ ] A/B 测试可看到对比指标（准确率、成本、延迟）

---

### 5.3 成本与用量 [P0]
- 全平台 token 实时消耗（按模型 / 商户 / 时段切分）
- 商户摊销：每商户 token 成本 / 等效收入
- 异常用量告警（单商户单日 token 超阈值）
- 单位成本：每会话成本 / 每成单成本
- **商户日 / 月 token 上限 + 超额降级**：
  - 80% → 告警
  - 100% → 自动切便宜模型
  - 150% → 暂停服务（或继续但人工审批，可配）

**验收标准**
- [ ] 实时消耗指标延迟 ≤ 5 分钟
- [ ] 商户摊销报表可导出 CSV
- [ ] 限额触发时商户端收到提示

---

### 5.4 AI 身份 / 规则管理 [P0]
- 全局系统 prompt 编辑器（含版本管理 + Diff）
- 行业模板 prompt（一期：防辐射服装）
- 商户级覆盖（特定商户的特化 prompt）
- 灰度发布（按商户百分比）
- **品牌口吻模板库**（CRUD + 上下架，一期 ≥ 4 种）

**验收标准**
- [ ] prompt 改动支持回滚
- [ ] 灰度发布可暂停 / 回滚
- [ ] 品牌口吻上下架对商户端立即生效

---

### 5.5 质量与安全 [P0]
- 差评 / 失败会话审核队列（来源：商户反馈、负面情绪、AI 不确定）
- AI 越权输出拦截规则：
  - 打折承诺（识别"X 折" + 数字）
  - 内部信息泄露
  - **功效宣传**（防辐射服装专门规则）
- Prompt 注入检测（识别"忘记之前指令"等模式）
- **对抗性消息识别**（套折扣 / 套情报 / 骚扰）→ 转人工 + 标记
- 评测集 + 回归：
  - 一期初始评测集 200 条
  - 每次 prompt / 模型变更 → 自动跑评测 → 对比基线
  - 回归通过率阈值（默认 ≥ 95%）

**验收标准**
- [ ] 越权输出被拦截后 AI 重生成（最多 2 次）
- [ ] 评测集运行 < 5 分钟
- [ ] 对抗性消息触发时商户端收到通知

---

### 5.6 知识库总控 [P0]
- 全部商户 KB 索引（搜索 / 筛选）
- 公共 KB（跨商户通用 FAQ，如"七天无理由"）
- 行业模板 KB（防辐射服装为首版）
- KB 治理：违规 / 过期 / 重复检测
- 单条 KB 可"提取为公共 / 行业模板"

**验收标准**
- [ ] 行业模板更新后已用模板的商户收到提示（不强制覆盖）
- [ ] 违规 KB 自动下线 + 通知商户

---

### 5.7 平台适配 [P0]
- 已接入平台清单（一期：淘宝 / 天猫 / 京东）
- 各平台 API 协议版本（含末次同步时间）
- 平台健康度（可用率、延迟、限流）
- 平台政策追踪：人工记录政策变化（最近一次更新、生效日期、应对措施）

**验收标准**
- [ ] 平台 API 可用率 < 95% 时告警
- [ ] 平台政策变化记录可被商户管理模块引用

---

### 5.8 运营基建 [P0]
- 内部账号 / 角色 / 权限
- 审计日志（谁改了什么）
- 计费引擎框架（一期搭骨架，二期开启积分逻辑）
- 财务报表：累计成本、按商户分布、未来计费占位

**验收标准**
- [ ] 审计日志至少保留 365 天
- [ ] 计费框架可在二期开关切换不重构

---

### 5.9 数据回流闭环 [P0]
- 好回答 → 公共 KB / 行业模板（脱敏后人工审核入库）
- 差回答 → 训练样本池 / 评测集
- 商户修正（"问题反馈"中的"正确答案"）→ 反向作用到该商户 prompt
- 跨商户聚合分析：高频问题、高频禁忌触发

**验收标准**
- [ ] 商户反馈进入训练池前必须经脱敏（PII 检测）
- [ ] 入公共 KB / 行业模板必须人工审核

---

### 5.10 告警 & SLA [P0]
- 系统级监控：API 可用率、延迟、错误率
- on-call 通知（钉钉 / 电话）
- SLA 看板：可用率、平均响应时长、月度故障时长
- **一键切回人工开关（三级粒度）**：
  - 单商户级
  - 单店铺级
  - 全局级（紧急熔断，所有商户切回原人工）

**验收标准**
- [ ] 全局熔断触发后 30s 内所有商户暂停 AI
- [ ] 切换记录留痕（谁切、何时、原因）

## 6. 关键流程

### 6.1 店铺接入流程
```
[商户端] 点击"接入店铺"
  → 选择平台（淘 / 天 / 京）
  → 跳转平台 OAuth 授权页
  → [客户输入账号密码授权]
  → 平台回调我方 callback URL
  → 拉取店铺信息 + token 入库（KMS 加密）
  → 触发商品自动同步任务（异步）
  → 商户端店铺列表显示"已接入"
  → 商户配置 AI 人设 + 工作模式
  → 进入沙盒测试 / 陪跑模式
```

### 6.2 对话主流程（全自动模式）
```
[平台消息推送] 客户消息到达
  → 我方接收（webhook）
  → 商户/店铺/会话路由
  → 检查启停开关 / 工作模式
  → 命中转人工规则？
      是 → 发过渡话术 + 推送至商户原客服 + 标 source=人工
      否 ↓
  → RAG 检索 KB + 拼装 prompt（含 AI 人设变量）
  → 模型路由 → 模型调用
  → 越权 / 合规检测
      违规 → 重生成（最多 2 次）→ 仍违规 → AI 兜底话术
      通过 ↓
  → 模拟人工节奏延迟
  → 整段发送至平台
  → 写入会话历史（source=AI）
  → 异步：成单归因检测、token 计费、监控埋点
```

### 6.3 陪跑→全自动切换流程
```
[陪跑期]
  消息 → AI 草稿 → 推送至商户端审核面板
  → 商户客服点击"通过 / 修改 / 拒绝"
  → 通过：发送 + source=AI（采纳）
  → 修改：发送修改后内容 + source=人工（草稿被否）
  → 拒绝：商户客服自行回复 + source=人工

[切换条件检查]（每日凌晨）
  - 沙盒测试通过 ≥ 200 条且准确率 ≥ 90% ✓
  - KB 完成度 ≥ 80% ✓
  - 培训完成（运营标记） ✓
  - 陪跑采纳率 ≥ 80%（连续 7 天） ✓
  → 商户端"切换至全自动"按钮亮起

[切换]
  商户老板点击切换 → 双重确认 → 模式切换 → 通知所有子账号
```

### 6.4 转人工流程
```
[消息进入]
  → 命中转人工规则（关键词 / 数量阈值 / 情绪 / 不确定度 / 对抗性）
  → AI 发过渡话术给客户
  → 标 source=人工
  → 推送通知到商户原客服平台（千牛 / 京麦），含会话摘要
  → 后续消息不再触发 AI（直到商户主动"还给 AI"）
```

### 6.5 问题反馈→KB 优化闭环
```
[商户运营] 在会话详情或沙盒看到错回答
  → 点击"反馈"
  → 选类型 + 填正确答案 + 标签 → 提交

[商户端处理]
  → 进入"问题反馈处理列表"
  → 商户运营选处理方式：
      A. 加入 FAQ → 跳转 FAQ 新建（带预填）
      B. 修正商品信息 → 跳转商品库
      C. 调整人设禁忌词 → 跳转 AI 人设
      D. 关闭（误报）
  → 处理完状态变"已处理"

[超管同步]
  → 反馈同步进入超管"训练样本池"
  → 脱敏后供超管筛选 → 加入评测集 / 公共 KB
```

### 7.2 二期占位字段说明

- `KnowledgeBase.scenario_scope` 一期 `[presale]`

### 7.3 多租户隔离
- 所有商户相关表强制 `merchant_id` 索引
- 跨租户查询 only in 超管端，需经审计日志

## 8. 合规与风险

### 8.1 数据合规（PIPL）

**红线**
- 注销请求触发后必须**真实删除**（不允许"前端删除、后台保留"）
- 法定保留范围（如交易记录最少 6 个月）外的所有数据须删除

**产品落点**
- 商户端"账户设置"提供"账号注销 + 数据导出"入口
- 注销流程：提交 → 7 天冷静期 → 真实删除 + 留 anonymized 统计
- 协议明确：训练用语料须经双重脱敏（PII 检测 + 人审）+ 商户协议明确授权
- 客户消息中的 PII（手机号 / 姓名 / 地址）在入超管训练池前自动脱敏

### 8.2 平台合规

**风险**
- 淘宝 / 天猫 / 京东对第三方 AI 客服有风控（限流 / 封号）
- 平台政策可能变化

**产品落点**
- 一期前置任务：开 3 个调研任务拉清三家平台 AI 客服 / 消息推送 API 政策
- 超管端"平台适配 / 平台政策追踪"模块持续维护
- 告警：商户在某平台被警告 / 封号 → 立即暂停该店铺 AI 服务
- 协议明确：平台合规由商户自负

### 8.3 品类合规（防辐射服装）

**红线**
- 不得承诺"100% 防辐射"、"孕妇必备"、"防止胎儿畸形"等医学功效
- 不得引用未认证的检测数据

**产品落点**
- 超管端预设品类禁忌话术包（防辐射服装专用）
- 质量审核规则：单独识别功效宣传越权
- 上线前为首批合作商定制审校话术包（人工 review）

### 8.4 商户协议必写清

- 数据使用授权（含语料反哺）
- 数据保留范围与期限
- AI 输出免责（最终决策权 / 事实核对责任）
- 平台合规由商户自负
- SLA 与故障赔付边界

## 9. 成功指标

### 9.1 一期 MVP 验收（首批合作商，3 个月内）

**Leading Indicators（早期）**
- 沙盒测试准确率 ≥ 90%（评测集 200 条 + 抽样人审）
- KB 完成度 ≥ 80%（商品同步率 + FAQ 覆盖率 + 必填字段填充率）
- 陪跑期 AI 草稿采纳率 ≥ 80%（连续 7 天）
- 接入店铺数（淘宝 + 天猫 + 京东，目标 ≥ 3）

**Lagging Indicators（上线后）**
- AI 独立处理会话占比 ≥ 70%（替代人工的核心证据）
- 客户盲测识别为人工占比 > 70%（不露 AI 痕迹）
- 客户负面反馈率（差评 + 投诉） ≤ 5%
- 成单转化率不低于人工对照组（同期同店历史均值）
- 等效节省人力 ≥ 7 人（10 人工的 70%）

### 9.2 业务指标
- 首批合作商月度续合作意愿（NPS / 续约调研）
- 二期商业化（积分制）启动条件：一期跑通 + ≥ 1 家新合作商接入意愿

### 9.3 合规指标
- PIPL 删除请求 100% 在 7 天内处理
- 平台违规事件 = 0
- 品类禁忌话术触发越权输出率 = 0（拦截后未发出）

## 10. 非目标（一期不做）

| 非目标 | 理由 |
|------|------|
| 售后场景（订单 / 物流 / 退换 / 价保） | 一期聚焦售前；售后涉及订单系统集成 + 风险更高，二期 |
| 抖音 / 小红书 / 微信 / 独立站 | 一期先把淘天京三家做透，避免多平台稀释精力 |
| 海外模型（Claude / GPT / Gemini） | 数据出境合规风险高，一期国产模型规避 |
| 主动外呼 / 主动触达 | 反骚扰合规复杂，二期单独立项 |
| 提供人工坐席兜底服务 | 我方不做客服外包，转人工指商户原客服 |
| 商户自由编辑 prompt | 中小商户写不好且会冲突底层规则，仅允许变量化模板 |
| 客户侧 AI 标识展示 | 决策为不露 AI 痕迹（产品形态决策） |
| 计费 / 积分 / 充值 | 一期免费试水，token 我方承担，二期再上 |
| 多语言 / 跨境 | 首批客户为中文国内市场 |
| 白标 / OEM | 一期不做，集中产品打磨 |
| 对外开放 API（让商户 ERP 调用） | 二期生态阶段再考虑 |

---

## 11. 二期路线占位

按预估优先级：

1. **售后场景接入**：订单 / 物流 / 退换 / 价保
2. **平台扩展**：抖音 → 小红书 → 微信公众号 / 小程序 → 独立站
3. **计费与商业化**：积分制、充值、超额扣费、套餐
4. **主动触达**：购物车提醒、收藏夹上新、活动召回
5. **海外模型 + 跨境**：评估数据出境合规后启用更强模型
6. **行业模板扩展**：3C、美妆、家居、食品类目模板
7. **白标 / OEM**：合作伙伴可定制品牌
8. **对外开放 API**：Webhook + REST API，对接商户 ERP / CRM
